## 一、项目概述

渠道开放平台（Partner Insight Hub）是一个面向外部渠道客户的自助服务门户网站。渠道客户通过平台提供的账号（邮箱 + 密码）登录后，可自助查看其在 Dida 平台的各项业务数据、系统集成状态、渠道市场表现、财务信息，并可查阅 Dida API 文档、通过内置 AI 助手解决技术问题，从而减少对运营（OP）人工支持的依赖，提升协作效率。

**技术栈**：React 18 + TypeScript + Vite + Ant Design 5，后端 Python FastAPI + SQLite，部署于 GitHub Pages（前端）+ 阿里云 ECS（后端）。

**访问地址**：`https://huangxiaozhen.github.io/Partner_Insight_Hub`

---

## 二、目标与收益

| # | 目标 | 预期收益 |
|---|------|----------|
| 1 | 数据自助化 | 渠道客户可自行查看有价率、准确率、失败率等关键指标，减少人工查询请求 |
| 2 | 日志自助化 | 订单各阶段日志客户可自行下载，减少争议，提升 OP 工作效率 |
| 3 | 时差问题兜底 | 存在时差的海外渠道在非工作时间可自助排查问题 |
| 4 | 提升 OP 专注度 | OP 从重复性数据查询中解放，专注于策略与更高价值工作 |
| 5 | 服务差异化 | 作为增值服务，提升渠道客户满意度和留存率 |

## 四、功能模块

平台登录后，侧边栏分为四大板块，共 14 个功能页面。

### 4.1 登录模块

渠道客户在登录页面输入邮箱账号和密码进行登录。

**页面样式**：与主系统保持一致的 Dida 品牌风格（navy #000947 主色调、白色卡片），展示 Dida Logo 及品牌名。

**登录验证逻辑**：

1. 客户输入邮箱 + 密码后点击登录，系统向后端 `/api/auth/login` 发起请求
2. 验证通过 → 跳转至主页（概览）
3. 异常提示：
   - 账号不存在 → 提示"账号不存在，请检查邮箱地址"
   - 密码错误 → 提示"密码错误，请重新输入"
   - 账号被禁用 → 提示"该账号已被禁用，请联系客户经理"

**其他**：
- 输入框仅显示灰色占位提示（邮箱示例：123@agoda.com，密码示例：Agoda123），不自动填充
- 支持记住登录状态（Remember Me）
- 密码重置功能：Demo 阶段暂不实现

### 板块四：支持与资源

#### 4.14 联系方式（Contact）

**侧边栏标签**：联系方式  
**功能描述**：集中展示 Dida 官方联系方式及客户我方对接人信息，支持客户自行维护对接人。

**分为两个区块：**

**① Dida 联系方式（只读，数据来源：数据库 dida_contacts 表）**

| 卡片 | 字段 |
|------|------|
| 客户经理 | 团队名称、邮箱、响应时间 |
| 技术支持 | 邮箱、紧急热线（+86 400-119-9777）、服务时间（7×24） |
| 商务合作 | 邮箱、响应时间 |
| 财务结算 | 邮箱、响应时间 |
| 客户服务热线 | 电话（+86 400-900-8811）、邮箱 |
| 紧急联系 | 紧急电话（+86 400-119-9777）、邮箱 |

以上字段均为只读，客户无法编辑。

**② 我方对接人（可编辑，数据来源：数据库 my_contacts 表）**

分为两类：运营对接人 和 商务对接人，各自支持：
- 新增：点击"添加"按钮，填写姓名、职位、邮箱、电话、微信号
- 编辑：点击铅笔图标进入编辑模式，修改后保存
- 删除：点击垃圾桶图标删除该联系人
- 数据持久化至后端 SQLite 数据库，通过 `GET/POST/PUT/DELETE /api/contacts/my` 接口读写

#### 4.15 Dida API 文档（Dida API）

**侧边栏标签**：Dida API  
**功能描述**：内嵌完整的 Dida API v2.0 技术文档，并提供 AI 智能问答助手，帮助渠道客户自助解决 API 集成技术问题。

**文档内容**：

| 模块 | 内容 |
|------|------|
| 平台概览 | API 版本（v2.0）、测试/生产环境地址、QPS 限制（默认 30）、数据格式（JSON，gzip 压缩）|
| 鉴权说明 | 请求 Header 需包含 ClientID + LicenseKey，附代码示例 |
| 预订流程 | 五步流程可视化：价格搜索 → 价格确认 → 创建预订 → 查询订单 → 取消预订，每步含接口端点 |
| 订单状态码 | 7 种状态（PreBook/Booked/Confirmed/Canceled/Failed/Pending/OnRequest），标注最终状态 |
| 错误码 | 定价类（2xxx）10 条 + 预订类（3xxx）8 条，含中文说明 |
| 接入上线流程 | 7 步 Go-Live 清单（签署 NDA → 获取开发环境 → 售后问卷 → 技术对接 → 测试认证 → 正式凭证 → 上线）|
| 技术规格 | 请求方式（HTTP POST）、URL 规范、多房间处理说明、取消二步流程（Pre-cancel + Confirm）|

**AI 智能助手（页面右下角悬浮聊天）**：
- 固定浮于页面右下角，点击 Navy 圆形按钮展开聊天面板（360×500px）
- 用户输入问题后，后端将问题与完整 Dida API 文档拼接为 Prompt，调用 Claude claude-sonnet-4-6 模型生成回答
- 支持多轮上下文对话，流式返回输出
- 仅限 Dida API 相关问题，不支持账户操作等实际业务

## 六、系统架构

```
浏览器（React + TypeScript, port 5173）
    ↓ Vite proxy /api → 
后端（Python FastAPI, port 8000, 0.0.0.0）

    ↓（AI 聊天）
```
